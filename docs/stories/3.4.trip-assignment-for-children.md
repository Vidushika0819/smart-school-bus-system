# Story 3.4: Trip Assignment for Children

## Status
Draft

## Story
**As a** parent,
**I want** to assign my children to available transportation routes,
**so that** they can be picked up and dropped off at school safely.

## Acceptance Criteria
1. Parents can view available trips/routes for their children's school
2. Parents can assign children to specific trips (pickup and drop-off)
3. Parents can view and manage all their children's trip assignments
4. System validates trip capacity and prevents over-assignment
5. Parents receive notifications about trip changes or issues
6. Children can only be assigned to trips that match their school location

## Dev Notes

### Previous Story Insights
Story 3.3 created the child information management system. This story extends that functionality by allowing parents to assign their registered children to available transportation trips/routes.

### Component Specifications
**Trip Assignment Data Model:**
- Fields: child, trip, assignmentType (pickup/dropoff/both), status, assignedDate
- Relationships: belongs to child and trip models
- Validation: One assignment per child per trip type, capacity limits
- Security: Parent can only assign their own children

**Trip Assignment Components:**
- TripAssignmentList: Grid view of all child-trip assignments with status
- TripSelector: Interface for selecting available trips by school/route
- AssignmentForm: Form for creating/modifying assignments with validation

**API Endpoints:**
- GET /api/trip-assignments - List parent's child assignments
- POST /api/trip-assignments - Create new assignment
- PUT /api/trip-assignments/:id - Update assignment
- DELETE /api/trip-assignments/:id - Cancel assignment
- GET /api/trips/available/:schoolId - Get available trips for school

### Technical Constraints
**Assignment Validation:**
- Children can only be assigned to trips serving their school
- Trip capacity limits must be respected
- No duplicate assignments for same child-trip-type combination
- Assignment conflicts (same child on multiple overlapping trips)

**Security Requirements:**
- Parents can only manage assignments for their own children
- Trip data should be filtered by school/location access
- Assignment modifications require proper authorization
- Audit trail for assignment changes

**Performance Considerations:**
- Efficient queries for available trips by school
- Optimized assignment lookups and capacity checks
- Caching for frequently accessed trip/route data
- Database indexing for assignment queries
