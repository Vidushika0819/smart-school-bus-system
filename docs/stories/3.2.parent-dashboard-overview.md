# Story 3.2: Parent Dashboard and System Overview

## Status
Draft

## Story
**As a** parent,
**I want** to view my dashboard and understand the system,
**so that** I can navigate the platform effectively.

## Acceptance Criteria
1. Parents see personalized dashboard after login
2. Dashboard displays overview of system functionality
3. Current active trips/routes are visible
4. Quick access to child management and trip assignment
5. Clear navigation to all parent features

## Dev Notes

### Previous Story Insights
Story 3.1 established parent authentication with login/registration. This story builds upon that foundation by creating a functional dashboard that parents will use to manage their children's transportation.

### Component Specifications
**ParentSidebar Component:**
- Vertical navigation menu for parent features
- Icons and labels for each section
- Active state highlighting
- Collapsible on mobile devices

**Dashboard Widgets:**
- Children Overview: Count and status summary
- Active Trips: Current trip assignments
- Quick Stats: Today's trips, pending actions, messages
- System Status: Available routes and school information

**Quick Actions Panel:**
- Add Child: Link to child management (future)
- View Trips: Link to trip assignment (future)
- Contact Admin: Direct messaging capability
- Update Profile: Parent profile management

### File Locations
**Frontend Files:**

### Technical Constraints
**Current Limitations:**
- No backend APIs for children/trips yet (will use mock data)
- Navigation links point to placeholder routes
- Real-time data will be implemented in future stories

**Future Integration Points:**
- Child management APIs (Story 3.3)
- Trip assignment system (Story 3.4)
- Communication system (Story 3.5)
