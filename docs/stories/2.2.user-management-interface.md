# Story 2.2: User Management Interface

## Status
Draft

## Story
**As an** admin user,
**I want** to manage all system users,
**so that** I can add, edit, and remove user accounts.

## Acceptance Criteria
1. Admin can view all users in the system with their roles and status
2. Admin can search and filter users by role, status, or email
3. Admin can create new user accounts with role assignment
4. Admin can edit existing user accounts and change roles

## Dev Notes

### Previous Story Insights
Story 2.1 established the admin dashboard foundation with role-based access control, navigation sidebar, and protected routing. User management interface will extend this foundation by adding comprehensive user administration capabilities.

### Data Models
**User Model:** [Source: docs/architecture.md#data-models-and-schema-changes]
- Existing User model with fields: email, password, role, status, createdAt, updatedAt
- Role enum: ["admin", "coordinator", "driver"]
- Status field: ["active", "inactive", "suspended"]
- Additional fields needed: lastLogin, loginAttempts, createdBy (for audit)

### API Specifications
**User Management Endpoints:** [Source: docs/architecture.md#admin-user-management-api]
- GET /api/admin/users - List all users with filtering and pagination
- GET /api/admin/users/:id - Get specific user details
- POST /api/admin/users - Create new user account
- PUT /api/admin/users/:id - Update user account
