# Story 6.1: Student Check-In Checkout System

## Status
Done

## Story
As a coordinator, I can mark each student's check-in when boarding the bus and check-out when leaving the bus, so that parents can track students' real-time travel status and ensure safety during school transportation.

## Acceptance Criteria
1. Coordinator can mark a student as checked in - system updates status to "On Board" and notifies parent via notification and email.
2. Coordinator can mark a student as checked out - system updates status to "Dropped Off" and notifies parent via notification and email.
3. System prevents duplicate check-in/out actions with appropriate error messages.
4. Parents can view student's current travel status (On Board / Dropped Off) in real-time on their dashboard.

### API Specifications
**Coordinator Protected Routes:**
- GET /api/coordinators/students-for-checkin - Get list of students assigned to trips for check-in/out (includes pending/checked_in)
- PUT /api/trip-assignments/:id/checkin - Update check-in status for specific assignment (requires coordinator role)
- PUT /api/trip-assignments/:id/checkout - Update check-out status for specific assignment (requires coordinator role)
- Responses include status updates and validation messages
- Role-based middleware: requireRole('coordinator') on all endpoints
[Source: docs/brownfield-architecture.md#Data Models and APIs]
